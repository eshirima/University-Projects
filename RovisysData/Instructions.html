Realm Class Wrapper Instructions

How to Use

Just copy the User.swift, ParentProject.swift and LineItems.swift to your main project. These files contain the classes which handle all of the Realm objects.

  • Object Set Up

    • Just use the helper function createNewUser as shown below

    let userObject = createNewUser(userName: String, password: String, userAdminStatus: Bool)

    This method creates a User object consisting of the passed data. The information stored in this object is then compared with the information stored in Realm for existence. If it does not exist, the information is saved in Realm and returned.

    Sample Usage

    let firstUser = createNewUser("people@yahoo.com", password: "hello", isAdmin: false)

    The same principle applies when trying to create a new parent project except the following helper function is called

    let parentProjectObject = createNewParentProject(projectName: String, projectOwner: String, projectSubmissionDate: NSDate, projectDescription: String)

    Sample Usage

    let firstProject = createNewParentProject("Capstone", projectOwner: firstUser.getUserName(), projectSubmissionDate: NSDate(), projectDescription: "Rovisys Expense Tracker")

    let secondProject = createNewParentProject("Site Surveying", projectOwner: firstUser.getUserName(), projectSubmissionDate: NSDate(), projectDescription: "Site survey of Aragon Electronics")

  • Link User to Project

    • As each project is created, they are automatically stored in a single private container called allUserProjects which acts as a bridge between the user and the parent project. With all the projects created, you can link them by calling linkUserToParentProject(currentUser: User) which takes user as its parameter.

    Sample Usage

    linkUserToParentProject(firstUser)

    When this method is invoked, the username is extracted using the getUserName() method and queried inside the parent project under the label submittedBy. If there does exist projects which have the current user's username, objects are returned consisting of the respective projects. These projects are then appended to the allUserProjects container.

    Remember that allUserProjects is our only interface between a user and parent project. With this container we can easily query between either sides using the stored attributes.

  • Print Methods

    • Thanks to Swift's print function taking AnyObject we can directly

    print(firstUser)

    This displays the user details in JSON format.

    The method searchForUserRelatedProjects() if invoked on the user object and returns a list of all the projects linked to the current user. To print out the information to the console, we wrap it inside the default Swift print function.

    Sample Usage

    print(firstUser.searchForUserRelatedProjects())

    printAllUsers() prints out all the users we have.

    Sample Usage

    print(firstUser.printAllUsers())

This is a simple guide but feel free to fiddle around to explore more functionalities.
